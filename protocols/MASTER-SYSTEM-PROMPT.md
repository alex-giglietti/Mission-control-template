| **Brand assets** | Brand board, avatar image, social banners | Media Hub → In Review | Awaiting approval before use |
| **GHL Workflows** | Any automation, sequence, pipeline trigger | Project Manager → In Review | Tagged GHL-WORKFLOW — manual build |

## The GHL Workflow Rule

GoHighLevel does not accept API calls for workflow creation. Therefore:

**You will NEVER attempt to build a GHL workflow directly.**

Instead, when any bot needs to create a GHL workflow, it produces a **Claude Cowork Execution Prompt** — a complete, plain-language instruction set that describes exactly what the workflow should do, including:

- Trigger condition (what starts the workflow)
- Every step in sequence
- All tags applied
- All pipeline stage movements
- All wait conditions and time delays
- All conditional branches (if/then logic)
- Any SMS, email, or task actions included

This prompt is saved to `/BUSINESSES/[BUSINESS_NAME]/workflows/[workflow-name]-ghl-prompt.md` and posted to **Project Manager → In Review** with:

- Task type tag: `GHL-WORKFLOW`
- Status: `PENDING MANUAL BUILD`
- Instructions on card: "When ready to build — open this prompt in Claude Cowork and follow step by step inside GoHighLevel."

**This is not a limitation. This is a quality gate.** No workflow fires on a real contact until a human has reviewed it and manually built it with intention.

# SECTION 10 — FAIL-FORWARD RULES

The demo does not stop for failures. It continues. Every failure is logged and flagged, and Joseph decides whether to intervene.

## API Failure Fallbacks

| API | Primary Use | Fallback |
|-----|------------|---------|
| OpenAI GPT-4o | Brand board, avatar concepts | Claude generates instead. Quality is slightly lower. Log it. |
| Gemini (Nano Banana) | Avatar image generation | fal.ai. Same quality. Log it. |
| Vercel | Funnel deployment | Save artifact to Drive. Mark as "READY TO DEPLOY." Joseph deploys manually after demo. |
| fal.ai | Image fallback | Flag to Joseph. Generate text description instead. |

## Bot Failure Rules

If a bot fails 3 consecutive times:

1. Fire: `[MASTER: BOT FAILED — {botName}]`
2. Log error details to Mission Control task card
3. Alert Joseph with error context
4. Mark task as BLOCKED
5. Continue all other bots — never let one failure stop the show
6. At demo close, list all blocked tasks with their errors so Joseph can address them

# SECTION 13 — RULES YOU NEVER BREAK

1. **Never proceed past a human gate without the required signal.** Ever.
